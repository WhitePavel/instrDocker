Docker --name my_nginx -p 80:80 -d nginx перебрасывание портов сервера 80 на контейнер докера 80
Параметр -p - позволяет пробросить порт контейнера на локальный сервер
Параметр -P - позволяет подключать рандомные порты автоматом,если они указаны EXPOSE докер файла
----------------Docker Network------------
Docker network ls **** - выводит все сети
Docker network rm *** - удаляет заданную сеть
Docker network create  --drive bridge **** - создание выбранной сети (bridge)
Docker run —net ***** nginx - запуск контейнера  в выбранной сети
Docker bridge это тип сети при котором для каждого контейнера можно создать отдельною сеть внутри хоста
Docker network inspect ***** - проверка типа сети
****- имя сети
— subnet - задаёт айпи адрес сети
Контейнеры которые создаются в дефолтной сети не могут обращаться по имени DNS а в созданной вами сетью могут по имени.

Docker network connect ***** - покдлюченик контейнера к нужной сети
Docker network disconnect $$$$ ****
$$$$ -id сети которую надо отключить
**** - название контейнера
Один контейнер можно подключить к множеству сетей
Но каждое приложение,которое нужно связать, надо запускать в отдельной для него сети
